## 시도해볼 플랫폼

[Expo](https://docs.expo.dev/)는 React Native용 도구와 서비스 프레임워크로, 앱 빌드, 배포, 반복 작업을 돕고, 웹 개발에서 널리 사용되는 미리보기 배포 워크플로를 활용하며, 개발 워크플로를 자동화하는 데 초점을 맞추고 있습니다. Expo를 사용하면 Xcode나 Android Studio를 한 번도 열지 않고도 React Native 앱을 빌드할 수 있으며, 해당 도구를 사용하고 싶을 때도 방해받지 않습니다.

[Ignite](https://github.com/infinitered/ignite)는 여러 React Native 보일러플레이트를 제공하는 스타터 킷 CLI입니다. 최신 버전인 Ignite Maverick은 상태 관리를 위한 MobX-State-Tree, React Navigation 및 기타 일반 라이브러리를 사용합니다. 화면, 모델 등에 대한 코드 제너레이터를 제공하며 기본적으로 Expo를 지원합니다. Ignite에는 커스텀 디자인, 테마 지원 및 테스트에 최적화된 컴포넌트 라이브러리도 포함되어 있습니다. 미리 구성된 기술 스택을 찾고 있다면 Ignite가 완벽한 선택일 수 있습니다.

## 네이티브 컴포넌트와 TurboModules 찾기, 만들기, 공유하기

React Native에는 콘텐츠, 도구, 튜토리얼, 그리고 네이티브 컴포넌트를 만드는 수천 명의 개발자 커뮤니티가 있습니다!

Core Components에서 원하는 것을 찾지 못하셨나요? [React Native Directory](https://reactnative.directory)를 방문하여 커뮤니티에서 만든 것들을 확인해보세요.

:::caution
이 문서는 레거시 API 세트를 참조하고 있으며 새로운 아키텍처를 반영하여 업데이트가 필요합니다.
:::
직접 네이티브 컴포넌트나 모듈을 만드는 데 관심이 있으신가요? 자신의 사용 사례에 맞는 모듈을 만들고 NPM과 GitHub을 통해 다른 사람들과 공유하면 React Native 생태계와 커뮤니티가 성장하는 데 도움이 됩니다! 네이티브 모듈([Android](legacy/native-modules-android.md), [iOS](legacy/native-modules-ios.md)) 및 네이티브 컴포넌트([Android](legacy/native-components-android.md), [iOS](legacy/native-components-ios.md)) 제작 가이드를 읽어보세요.
