React Native에 기여해 주셔서 감사합니다! 이슈에 댓글을 달고 분류하는 것부터 풀 리퀘스트를 리뷰하고 보내는 것까지, 모든 기여를 환영합니다.
우리는 메인 React Native GitHub 저장소를 넘어서는 [파트너, 핵심 기여자, 커뮤니티의 활발하고 포용적인 생태계](https://github.com/facebook/react-native/blob/main/ECOSYSTEM.md)를 구축하는 것을 목표로 합니다.

[오픈 소스 가이드](https://opensource.guide/) 웹사이트에는 오픈 소스 프로젝트를 운영하고 기여하는 방법을 배우고자 하는 개인, 커뮤니티, 기업을 위한 리소스 모음이 있습니다.

기여자와 오픈 소스 입문자 모두에게 다음 가이드가 특히 유용할 것입니다.

- [오픈 소스에 기여하는 방법](https://opensource.guide/how-to-contribute/)
- [환영하는 커뮤니티 만들기](https://opensource.guide/building-community/)

### 행동 강령

모든 기여자는 [행동 강령](https://github.com/facebook/react-native/blob/HEAD/CODE_OF_CONDUCT.md)을 준수해야 한다는 점을 다시 한번 상기해 주세요.

## 버전 관리 정책

React Native의 버전 관리를 완전히 이해하려면 [버전 관리 정책](/docs/releases/versioning-policy) 페이지를 확인하시길 권장합니다.
해당 페이지에서는 어떤 버전의 React Native가 지원되는지, 얼마나 자주 릴리즈되는지, 상황에 따라 어떤 버전을 사용해야 하는지를 설명합니다.

## 기여 방법

코드 기여를 바로 시작하고 싶다면 상대적으로 제한된 범위의 버그를 포함한 [good first issues](https://github.com/facebook/react-native/labels/good%20first%20issue) 목록이 있습니다.
경험이 쌓이고 React Native 발전에 대한 헌신을 보여주면, 저장소에서 이슈 관리 권한을 부여받을 수 있습니다.

코드 한 줄 작성하지 않고도 기여할 수 있는 다른 방법들이 있습니다. 도움이 될 수 있는 몇 가지를 소개합니다.

1. **열린 이슈에 답변하고 처리하기.**

   매일 많은 이슈가 들어오며, 일부는 필요한 정보가 부족할 수 있습니다. 이슈 템플릿을 작성하는 과정을 안내하거나, 추가 정보를 요청하거나, 문제 설명과 일치하는 기존 이슈를 안내하는 방식으로 도움을 줄 수 있습니다.
   이 프로세스에 대한 자세한 내용은 [GitHub 이슈 분류](/contributing/triaging-github-issues) 페이지에서 다루고 있습니다.

2. **문서 풀 리퀘스트 리뷰하기.**

   [문서 업데이트](https://github.com/facebook/react-native-website/pulls) 리뷰는 철자와 문법을 확인하는 것만큼 간단할 수 있습니다.
   문서에서 더 잘 설명될 수 있는 상황을 발견하면, 대부분의 문서 페이지 상단의 **Edit**을 클릭하여 직접 기여를 시작하세요.

3. **테스트 계획 작성 돕기.**

   메인 저장소에 보내진 일부 풀 리퀘스트에는 적절한 테스트 계획이 없을 수 있습니다. 테스트 계획은 리뷰어가 변경 사항이 어떻게 테스트되었는지 이해하는 데 도움이 되며, 기여가 수락되는 시간을 단축할 수 있습니다.

이러한 작업 각각은 매우 영향력이 크며, 메인테이너들은 여러분의 도움에 크게 감사할 것입니다.

### 개발 프로세스

우리는 GitHub 이슈와 풀 리퀘스트를 사용하여 커뮤니티의 버그 리포트와 기여를 추적합니다. Meta 엔지니어들의 모든 변경 사항은 Meta의 내부 소스 관리 시스템과의 브리지를 통해 [GitHub](https://github.com/facebook/react-native)에 동기화됩니다. 커뮤니티의 변경 사항은 GitHub 풀 리퀘스트를 통해 처리됩니다.

GitHub에서 변경 사항이 승인되면, 먼저 Facebook의 내부 소스 관리 시스템으로 임포트되어 Facebook의 코드베이스를 대상으로 테스트됩니다. Facebook에서 머지된 후, 변경 사항은 Facebook의 내부 테스트를 통과하면 단일 커밋으로 GitHub에 다시 동기화됩니다.

기여 프로세스에 대한 자세한 내용은 다음 문서에서 확인할 수 있습니다.

- [GitHub 이슈 분류](/contributing/triaging-github-issues)
- [풀 리퀘스트 관리](/contributing/managing-pull-requests)

또한 설정을 도와줄 활발한 기여자 커뮤니티도 있습니다. [@ReactNative](https://twitter.com/reactnative)를 통해 React Native 팀에 연락하실 수 있습니다.

### 저장소

메인 저장소에는 React Native 프레임워크 자체가 포함되어 있으며, 버그 리포트를 추적하고 풀 리퀘스트를 관리하는 곳입니다.

익숙해지면 좋을 다른 저장소들도 있습니다.

- **React Native 웹사이트**: 문서를 포함한 웹사이트의 소스 코드가 [이 저장소](https://github.com/facebook/react-native-website)에 있습니다.
- **릴리즈** 논의는 [이 토론 저장소](https://github.com/reactwg/react-native-releases/discussions)에서 진행됩니다.
- **릴리즈의 Changelog**는 [여기](https://github.com/facebook/react-native/blob/main/CHANGELOG.md)에서 확인할 수 있습니다.
- **React Native에 관한 토론**은 [Discussions and Proposals](https://github.com/react-native-community/discussions-and-proposals) 저장소에서 이루어집니다.
- **React Native의 새로운 아키텍처에 관한 토론**은 [React Native New Architecture Working Group](https://github.com/reactwg/react-native-new-architecture) 저장소에서 이루어집니다.
- **고품질 플러그인**은 [React Native Directory](https://reactnative.directory) 웹사이트에서 찾을 수 있습니다.

이 저장소들을 둘러보면 React Native 오픈 소스 프로젝트가 어떻게 관리되는지 파악하는 데 도움이 됩니다.

## GitHub 이슈

GitHub 이슈는 버그를 추적하는 데만 사용합니다. 이슈 처리 프로세스는 [이슈 분류 페이지](/contributing/triaging-github-issues)에 문서화되어 있습니다.

### 보안 버그

Meta에는 보안 버그의 안전한 공개를 위한 [포상금 프로그램](https://www.facebook.com/whitehat/)이 있습니다. 해당 경우에는 해당 페이지에 설명된 프로세스를 따르고 공개 이슈를 제출하지 마세요.

## 문서 기여

React Native 문서는 React Native 웹사이트 저장소의 일부로 호스팅됩니다. 웹사이트는 [Docusaurus](https://docusaurus.io/)를 사용하여 빌드됩니다. 문서에서 변경하고 싶은 내용이 있다면, 웹사이트 대부분의 페이지 오른쪽 상단에 있는 "Edit" 버튼을 클릭하여 시작할 수 있습니다.

새로운 기능을 추가하거나 동작 방식을 변경하는 경우, 변경 사항을 반영하도록 문서를 업데이트해 주시기 바랍니다.

### 블로그 기여

React Native 블로그는 [블로그의 Markdown 소스](https://github.com/facebook/react-native-website/tree/HEAD/website/blog)에서 생성됩니다.

React Native 블로그용 기사를 작성하기 전에 React Native 웹사이트 저장소에 이슈를 열거나 [트위터의 @ReactNative](https://twitter.com/reactnative)로 태그하여 메인테이너의 승인을 받으세요.
대부분의 경우, 자신의 블로그나 다른 미디어에 기사를 공유하는 것이 좋을 수 있습니다. 그래도 블로그에 적합한 기사인 경우를 위해 물어보는 것은 좋습니다.

웹사이트 기여에 대한 자세한 내용은 `react-native-website` 저장소의 [Readme 파일](https://github.com/facebook/react-native-website#-contributing)을 참조하시기 바랍니다.

## 코드 기여

React Native에 대한 코드 수준의 기여는 일반적으로 [풀 리퀘스트](https://help.github.com/en/articles/about-pull-requests) 형태로 이루어집니다. 저장소를 포크하고 로컬에서 변경을 진행하면 됩니다.

### 단계별 가이드

코드를 기여할 준비가 되었다면 [첫 번째 풀 리퀘스트를 보내는 단계별 가이드](/contributing/how-to-open-a-pull-request)를 확인하거나, [코드 기여 방법](/contributing/how-to-contribute-code) 페이지에서 자세한 내용을 읽어보세요.

### 테스트

테스트는 코드베이스에 회귀가 발생하는 것을 방지하는 데 도움이 됩니다. GitHub 저장소는 CircleCI를 사용하여 지속적으로 테스트되며, 그 결과는 [커밋](https://github.com/facebook/react-native/commits/HEAD)과 풀 리퀘스트의 Checks 기능을 통해 확인할 수 있습니다.

테스트 실행 및 작성에 대한 자세한 내용은 [테스트 실행 및 작성 방법](/contributing/how-to-run-and-write-tests) 페이지에서 확인할 수 있습니다.

## 커뮤니티 기여

React Native에 대한 기여는 GitHub에 국한되지 않습니다. 블로그 게시물 작성, 컨퍼런스에서 발표, 또는 트위터에서 생각을 공유하고 [@ReactNative](https://twitter.com/reactnative)를 태그하는 방식으로 React Native 사용 경험을 나누어 다른 사람들을 도울 수 있습니다.
